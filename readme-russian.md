# ideasgame: README (рус.)

*Агент‑ориентированная симуляция распространения идей через призму психологии личности*


## Обзор


**ideasgame** моделирует, как идеи распространяются, трансформируются и формируют сообщества. В основе движка — сочетание:
- **Личностных черт** (OCEAN + интеллект);  
- **Динамики идей** (содержание, эмоциональный оттенок, искажение);  
- **Социальных механизмов** (влияние, адаптация, эволюция).

## Ключевые компоненты

### 1. Класс `Idea`
Описывает распространяемую идею со следующими атрибутами:
- `core`: текстовая суть идеи;  
- `effects`: влияние на параметры агентов (напр., `{'C': +8, 'A': +5}`);  
- `complexity`: 0–100 (определяет степень искажения при передаче);  
- `emotion`: эмоциональный оттенок (напр., «благоговение»);  
- `weight`: сила идеи в сообществе (растёт с распространением).

### 2. Класс `Agent`
Каждый агент обладает 6 чертами (шкала 0–100):
- **O** (Открытость) — восприимчивость к новому опыту;  
- **C** (Добросовестность) — следование правилам, надёжность;  
- **E** (Экстраверсия) — социальная активность;  
- **A** (Доброжелательность) — эмпатия, доброта;  
- **N** (Нейротизм) — чувствительность к угрозам;  
- **I** (Интеллект) — критическое мышление.


**Основные методы**:
- `calculate_social_weight()`: вычисляет влияние (E×0,4 + C×0,3 + I×0,3);  
- `transmit_idea()`: передаёт идею с искажением и адаптацией;  
- `adapt_idea()`: изменяет идею на основе черт источника и получателя;  
- `receive_idea()`: применяет эффекты с фильтрацией (напр., высокий C сопротивляется негативным изменениям);  
- `evolve_idea()`: обновляет сложность и эмоциональный оттенок идеи после передачи.


### 3. Класс `Community`
Управляет популяцией агентов и симуляцией:
- `add_idea()`: добавляет новые идеи в систему;  
- `step()`: выполняет один цикл симуляции (выбор источника → получателя → передача → обновление);  
- `update_history()`: фиксирует средние значения черт;  
- `plot_history()`: визуализирует динамику параметров.


## Рабочий процесс симуляции


1. **Инициализация**  
   - Создать `Community(size=N)`;  
   - Добавить начальные экземпляры `Idea`.


2. **Цикл (`step()`)**  
   а. Случайный **агент‑источник** (должен иметь идеи в убеждениях);  
   б. Случайная **идея** из убеждений источника;  
   в. **Получатель** выбирается по взвешенной вероятности (E источника × A получателя);  
   г. **Передача**:  
      - Применяется искажение (зависит от сложности идеи и I источника);  
      - Идея адаптируется под черты источника и получателя;  
      - Получатель обновляет свои черты и принимает идею;  
      - Идея эволюционирует (меняется сложность/эмоциональный оттенок).  
   д. Средние значения черт записываются в историю.


3. **Завершение**  
   - После `num_steps` выводятся графики и статистика.


## Результаты симуляции


1. **Лог консоли**  
   ```
   Шаг 0: средний O=49,4, C=48,9, E=50,0  
   ...  
   Симуляция завершена (100 шагов).
   ```


2. **Графики**  
   - Линейный график средних значений OCEAN+I по шагам.


3. **Итоговая статистика**  
   - Средние, минимальные и максимальные значения каждой черты по всем агентам;  
   - Уровень принятия идей (напр., *«Соблюдай завет»: 22/30 агентов*);  
   - Топ‑5 социальных лидеров (по `social_weight`).


## Использование


1. **Установка зависимостей**  
   ```bash
   pip install matplotlib
   ```

2. **Запуск симуляции**  
   ```bash
   python main.py
   ```

3. **Настройка**  
   - Измените `Community(size)` и начальные идеи в блоке `__main__`;  
   - Настройте параметры идей: `effects`, `complexity`, `emotion`.


## Параметры и настройка


- **Черты агентов**: инициализируются случайно в заданных диапазонах (напр., O: 30–70);  
- **Искажение**: `±0,1 × (complexity / 100)`, снижается за счёт I источника, растёт из‑за N получателя;  
- **Правила адаптации**:  
  - Высокий A (источник) → смягчает текст идеи;  
  - Низкий O (получатель) → снижает сложность идеи;  
  - Высокий N (получатель) → усиливает эмоциональный оттенок;  
- **Эволюция идеи**: `weight × 1,01` за каждую передачу; сложность ±5 % в зависимости от O источника.

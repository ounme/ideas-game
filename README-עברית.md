# BELIVEME: README

*סימולציה מבוססת-סוכנים של הפצת רעיונות דרך פסיכולוגיית האישיות*

## סקירה כללית

**BELIVEME** מדגימה כיצד רעיונות מתפשטים, משתנים ומעצבים קהילות. המנוע משלב:
- **תכונות אישיות** (OCEAN + אינטליגנציה);  
- **דינמיקה של רעיונות** (תוכן, גוון רגשי, עיוות);  
- **מנגנונים חברתיים** (השפעה, הסתגלות, אבולוציה).

## רכיבים עיקריים

### 1. מחלקת `Idea`
מייצגת קונספט שניתן להפצה עם:
- `core`: מהות טקסטואלית;  
- `effects`: השפעות על פרמטרים (למשל, `{'C': +8, 'A': +5}`);  
- `complexity`: 0–100 (משפיע על עיוות במהלך שידור);  
- `emotion`: גוון רגשי (למשל, "יראת כבוד");  
- `weight`: כוחו של הרעיון בקהילה (גדל עם אימוץ).

### 2. מחלקת `Agent`
לכל סוכן יש 6 תכונות (0–100):
- **O** (Openness) — קבלתיות לחוויות חדשות;  
- **C** (Conscientiousness) — ציות לכללים, אמינות;  
- **E** (Extraversion) — פעילות חברתית;  
- **A** (Agreeableness) — אמפתיה, נדיבות;  
- **N** (Neuroticism) — רגישות לאיומים;  
- **I** (Intelligence) — חשיבה ביקורתית.

**מתודות ליבה**:
- `calculate_social_weight()`: מחשבת השפעה (E×0.4 + C×0.3 + I×0.3);  
- `transmit_idea()`: שולחת רעיון עם עיוות והסתגלות;  
- `adapt_idea()`: משנה רעיון בהתבסס על תכונות מקור/מקבל;  
- `receive_idea()`: מיישמת השפעות עם פילטרים (למשל, C גבוה מתנגד לשינויים שליליים);  
- `evolve_idea()`: מעדכנת מורכבות ורגש של הרעיון לאחר שידור.

### 3. מחלקת `Community`
מנהלת את אוכלוסיית הסוכנים והסימולציה:
- `add_idea()`: מזריקה רעיונות חדשים למערכת;  
- `step()`: מריצה מחזור סימולציה אחד (בחירת מקור → מקבל → שידור → עדכון);  
- `update_history()`: רושמת ערכי תכונות ממוצעים;  
- `plot_history()`: מציגה מגמות פרמטרים.

## זרימת הסימולציה

1. **אתחול**  
   - יצירת `Community(size=N)`;  
   - הוספת מופעי `Idea` התחלתיים.

2. **מחזור (`step()`)**  
   א. **סוכן מקור** אקראי נבחר (חייב להחזיק רעיונות);  
   ב. **רעיון** אקראי נבחר מהאמונות של המקור;  
   ג. **מקבל** נבחר באמצעות הסתברות משוקללת (E של המקור × A של המקבל);  
   ד. **שידור**:  
      - עיוות מיושם (מבוסס על מורכבות הרעיון ו-I של המקור);  
      - רעיון מותאם לתכונות מקור/מקבל;  
      - מקבל מעדכן תכונות ומאמץ רעיון;  
      - רעיון מתפתח (מורכבות/רגש משתנים).  
   ה. ממוצעי תכונות נרשמים.

3. **סיום**  
   - לאחר `num_steps`, הצגת תוצאות והדפסת סטטיסטיקות.

## פלט

1. **יומן קונסול**  
   ```
   Шаг 0: средний O=49.4, C=48.9, E=50.0  
   ...  
   Симуляция завершена (100 шагов).
   ```

2. **גרפים**  
   - תרשים קווים של ערכי OCEAN+I ממוצעים על פני שלבים.

3. **סטטיסטיקות סופיות**  
   - ממוצע/מינימום/מקסימום לכל תכונה על פני סוכנים;  
   - שיעורי אימוץ רעיונות (למשל, *"Соблюдай завет": 22/30 סוכנים*);  
   - 5 המובילים החברתיים המובילים (לפי `social_weight`).

## שימוש

1. **התקנת תלויות**  
   ```bash
   pip install matplotlib
   ```

2. **הרצת סימולציה**  
   ```bash
   python main.py
   ```

3. **התאמה אישית**  
   - שינוי `Community(size)` ורעיונות התחלתיים בבלוק `__main__`;  
   - התאמת ערכי `effects`, `complexity` ו-`emotion` של רעיונות.

## פרמטרים וכוונון

- **תכונות סוכנים**: אתחול אקראי בטווחים (למשל, O: 30–70);  
- **עיוות**: `±0.1 × (complexity / 100)`, מופחת על ידי I של המקור, מוגבר על ידי N של המקבל;  
- **כללי הסתגלות**:  
  - A גבוה (מקור) → מרכך טקסט ליבה;  
  - O נמוך (מקבל) → מוריד מורכבות;  
  - N גבוה (מקבל) → מעצים רגש;  
- **אבולוציה של רעיונות**: `weight × 1.01` לכל שידור; מורכבות ±5% מבוססת על O של המקור.
